# クイックスタート
https://cloud.google.com/appengine/docs/standard/nodejs/quickstart?hl=ja
[請求先アカウントの作成](https://cloud.google.com/billing/docs/how-to/modify-project?hl=ja)

[Cloud SDKのインストール](https://cloud.google.com/sdk/docs/install?hl=ja#other_installation_options)

- Cloud Build APIを有効にする
- 

Homebrewを使う場合:
```
% brew install google-cloud-sdk
```

初期化
```
# googleアカウントにログイン
% gcloud auth login
```

Configurationを増やす場合
https://qiita.com/toshi0607/items/90cbbacdfae6c4282c0a

Configurationを切り替える場合
https://qiita.com/sonots/items/906798c408132e26b41c

```
% gcloud config configurations create [設定名]
% gcloud config set project [PROJECT_ID]
% gcloud config set account [ログインしたアカウント]
% gcloud config configurations activate [設定名]
```


App Engineの初期化
```
% gcloud app create
```

リージョン
https://cloud.google.com/compute/docs/regions-zones?hl=ja
- asia-northeast1: 東京
- asia-northeast2: 大阪

サンプルをクローン
```
% git clone https://github.com/GoogleCloudPlatform/nodejs-docs-samples
% cd nodejs-docs-samples/appengine/helloworld/standard
% npm install
% npm start
# http://localhost:8080 を確認する
```


デプロイする
```
% gcloud app deploy --project=[PROJECT_ID]
```

エラーになる。

```
ERROR: (gcloud.app.deploy) NOT_FOUND: Unable to retrieve P4SA: [****@gcp-gae-service.iam.gserviceaccount.com] from GAIA. Could be GAIA propagation delay or request from deleted apps.
```
https://stackoverflow.com/questions/66528149/deploying-django-on-google-app-engine-error-gcloud-app-deploy-not-found
数分待ってからもう一度行うと、成功する。
おそらく、エラー文にあるように、デプロイ用のアカウントの作成がまだ完了していないと思われる

プロジェクトを削除する
https://console.cloud.google.com/iam-admin/projects?hl=ja

Configurationを削除する
```
% gcloud config configurations activate default
% gcloud config configurations delete [設定名]
```

## プロジェクトの作成
プロジェクトを作成する。
https://console.cloud.google.com/projectselector2/home/dashboard?hl=ja

Clooud Build APIを有効にする
- https://console.cloud.google.com/home/dashboard?hl=ja に移動する
- 作成したプロジェクトが選択された状態にする
- 左上のハンバーガーメニュー＞APIとサービス＞ライブラリ
- Cloud Build APIを検索し、有効化する
- Configurationを作成する
- App Engineを初期化

## Node.jsを使用したウェブサービスの作成
```
% cd path/to/project
% npm install express-generator -g
% express --view=pug
% npm install
% npm audit fix --force
% npm start
# http://localhost:3000 を確認する
```

app.yamlの作成
```
runtime: nodejs14
```
[app.yaml 構成ファイル  |  Node.js 用 App Engine スタンダード環境に関するドキュメント  |  Google Cloud](https://cloud.google.com/appengine/docs/standard/nodejs/config/appref)
