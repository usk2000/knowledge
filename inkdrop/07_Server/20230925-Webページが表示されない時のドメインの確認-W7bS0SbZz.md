# Webページが表示されない時のドメインの確認


引越しで、ドメインが新しいところを向かない不具合

DNSをチェック
[奥が深いDNSサーバーとdigコマンド - Qiita](https://qiita.com/hypermkt/items/610b5042d290348a9dfa)
[DIGを使用したDNSサーバの確認方法について | セキュリティ専門企業発、ネットワーク・ログ監視の技術情報 - KnowledgeStare（ナレッジステア）](https://www.secuavail.com/kb/windows-linux/linux-dig-dns/)

[digコマンドでDNSレコードを取得する | アナライズギア開発ブログ](https://analyzegear.co.jp/blog/669)


https://www.cman.jp/network/support/nslookup.html
ここからのページだと
```
ongaku-hosekibako.com.	172445	IN	NS	mdns2.fc2.com.
ongaku-hosekibako.com.	172445	IN	NS	mdns1.fc2.com.
```
→うまく表示される

コマンドからだと
```bash
% dig [domain]
;; OPT PSEUDOSECTION:
; EDNS: version: 0, flags:; udp: 512
;; QUESTION SECTION:
;ongaku-hosekibako.com.		IN	NS

;; AUTHORITY SECTION:
ongaku-hosekibako.com.	10670	IN	SOA	ns.ongaku-hosekibako.com. mail.ongaku-hosekibako.com. 1578506 28800 7200 604480 86400

```
→表示されない


原因と解決方法
- ドメインを持っているサービス（今回はさくらインターネット）のなかのネームサーバの設定がおかしかった
- NSをさくらインターネットに向けたら直った
