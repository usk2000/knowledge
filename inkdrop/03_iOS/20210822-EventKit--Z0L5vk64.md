## 権限をリクエストし取得
カレンダー・リマインダー使用のリクエストを行う

事前に`Info.plist`に以下のキーを追加する。
- NSCalendarsUsageDescription: カレンダーを使う場合
- NSRemindersUsageDescription: リマインダーを使う場合
- NSContactsUsageDescription: カレンダーUI(EventKit UI)を使う場合

リクエストを行う

```swift
let store = EKEventStore()
        
store.requestAccess(to: .reminder) { granted, error in
                                    
}
```

取得を行う
https://developer.apple.com/documentation/eventkit/retrieving_events_and_reminders
```swift
        let predicate = store.predicateForReminders(in: nil)
        store.fetchReminders(matching: predicate) { reminders in
            guard let reminders = reminders else {
                return
            }
            
            reminders.forEach { reminder in
                debugPrint(reminder)
            }
            
        }
```

リストの取得
https://stackoverflow.com/questions/16724822/create-new-reminders-list-with-eventkit
https://developer.apple.com/documentation/eventkit/eksource
```swift
        let calendars = store.calendars(for: .reminder)

```

カレンダーの作成
https://developer.apple.com/documentation/eventkit/ekcalendar/1507516-init
```swift
let calendar = EKCalendar(for: .reminder, eventStore: store)
let source = store.sources.first(where: { $0.sourceType == .calDAV })!
calendar.title = "Title"
calendar.source = source

do {
                try store.saveCalendar(appCalendar, commit: true)
            } catch let error {
                debugPrint(error)
            }
```

カレンダーを指定して取得
```swift
let predicate = store.predicateForReminders(in: [calendar])
store.fetchReminders(matching: predicate) { reminders in
                                           
}
```

リマインダーの追加と保存
```swift
let reminder = EKReminder(eventStore: store)
reminder.title = "Item"
reminder.calendar = calendar
reminder.notes = "Memo"
do {
  try store.save(reminder, commit: true)
} catch let error {
}
```

リマインダーの削除
```swift
do {
  try store.remove(reminder, commit: true)
} catch let error {
}
```

## 結論
Reminder APIはDBとして使えそうではあるが、やりたいことに対して自由度や挙動に難があるので
CloudKitのままの方が良さそう。
