# データ保持ポリシー


- System Key-Value Store:
  - iOS: UserDefaults
  - Android: SharedPreferences
- File on Disk
  - iOS: only inner storage
    - Documents, Library/Cache, tmp
  - Android: inner and external storage
  - ex: json file or image file
- キーチェーン
  - iOS: Keychaine
  - Android: Keystore
- ライブラリ・フレームワーク
  - iOS: CoreData, Realm, SQLite etc
  - Android: Realm, SQLite etc

UserDefautlsなどは、構造化されていないデータなどは保持できる。

ファイル保存は、構造化されているデータを保持できる。軽量なjsonファイルなど。
アプリにとって常に必要なファイルか、都度の取得で良いかによって場所を分ける。
Androidの内部ストレージはあまり容量スペースがない場合が多いが保存しやすい。外部ストレージは容量が多くなりやすい反面、アクセスしにくい。

画像についてはキャッシュはシステムに任せる。
キーチェーンは情報を永続化できる。

ライブラリ、CoreDataなどのフレームワークは特段の必要性がなければ使わない。
CoreData, Realmなどのデータベースは便利（フィルタ・大容量など）な反面、学習コスト高く・メンテナンス性に難がある。単なるキャッシュとしては使うべきでない。
