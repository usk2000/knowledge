# AppAttest
正しいアプリであることの証明

## Establish Your App's Integrity
https://developer.apple.com/documentation/devicecheck/establishing_your_app_s_integrity
```
You can’t rely on your app’s logic to perform security checks on itself because a compromised app can falsify the results. Instead, you use the shared instance of the DCAppAttestService class in your app to create a hardware-based, cryptographic key that uses Apple servers to certify that the key belongs to a valid instance of your app. Then you use the service to cryptographically sign server requests using the certified key. Your app uses these measures to assert its legitimacy with any server requests for sensitive or premium content.
```

> 侵害されたアプリは結果を改ざんする可能性があるため、アプリのロジックに依存してセキュリティチェックを実行することはできません。 代わりに、アプリでDCAppAttestServiceクラスの共有インスタンスを使用して、ハードウェアベースの暗号化キーを作成します。このキーは、Appleサーバーを使用して、キーがアプリの有効なインスタンスに属していることを証明します。 次に、このサービスを使用して、認証されたキーを使用してサーバー要求に暗号で署名します。 アプリはこれらの手段を使用して、機密性の高いコンテンツやプレミアムコンテンツに対するサーバーのリクエストに対して正当性を主張します。
>

### 再インストール時にはキーを再生成
> 取得したキーは基本保持し続けますが、再インストール時・デバイス移行・バックアップからの復元の時には取得し直さなければなりません。
> キーの取得はこれらのイベント以外では制限する必要があります。制限することで、より不正を見つけやすくなります。
>
