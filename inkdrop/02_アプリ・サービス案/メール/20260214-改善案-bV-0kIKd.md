# 改善案



## Datadog

- Firebase Performance Monitoring: ネットワークリクエスト・起動時間などのパフォーマンス系
- Firebase Analytics: ユーザ行動
- 技術ログ: 現在なし→Datadogの実装を勧めたい

### 利点

- 不具合が起きた時のユーザの詳細な動きが追える
- アプリだけでなく、バックエンドにもまたいで調査が可能

### Datadogの欠点

- 従量課金
→これが効いてくるので、提案はなし。


## API仕様をOpenAPIで管理

- 現在のスプレッドシート管理から、OpenAPIへ移行

### 利点
- 仕様のずれがかなり少なくなる。ずれがあるとしたら、パラメータ・レスポンスの軽微な運用の方法
- 通信部分は自動生成できる
- API部分の変更が簡単になる。ただし、ロジック・UI部分は引き続き変更は必要。

### 欠点
- API Client部分の作り直しだが、影響は中くらい

## WebKit deprecated対応

  - WKWebViewConfiguration.processPoolのDeprecated対応
  - WKWebViewConfiguration.websiteDataStoreを使用する

### 利点
- 今後非対応になるので、今のうちに対応するとトラブルが少ない

### 欠点
- WebViewを使うところの挙動をテストしなおし


## 画像読み込みライブラリを変える
現在のアニメーションgifを読み込むライブラリが古く更新がされていないので、最新のライブラリに変更する

### 利点
- 今後のOSアップデートに追従する

### 欠点
- 差し替えとテスト

## 一部画面のSwiftUI化
- 現在はUIKitと呼ばれる古いUIフレームワークを使用している
- 新しいUIフレームワークに、部分的に変更する。たとえばInfo画面と付随する画面

### 利点
- レイアウトのコードが少なくなるので、開発速度が速くなる
- アプリを動かした時の画面表示も早くなる

### 欠点
- 複雑な画面はSwiftUIにしづらい
