# アドバイス機能


**ラフ評価**は「毎回すぐ返す・安い・UXを軽くする」ために使い、  
**詳細評価**は「Pronunciation Map更新・弱点特定・精密な学習」に使う、という役割分担が自然です。Accent Fitのループは Prompt → Speak → Score → Improve なので、この2段構えはかなり相性が良いです

## 1. Ruleによるアドバイスでできること

Apple Speechだけを使う場合、基本的には以下の入力を使います。

- Prompt text
- Recognized text
- Word timestamp
- Recognition confidence
- Missing / extra / replaced words
- 発話時間
- 単語ごとの間隔
- 無音区間

ここから作れるRule adviceは、主に **発音そのものというより、発話結果から推定できる改善アドバイス** です。

## Rule adviceの例

### 1. 単語抜け

Prompt:

> I really like this one.

Recognized:

> I like this one.

Advice:

> **“really” が弱くなっています。少し長めに、はっきり出してみましょう。**

これはかなり実用的です。

### 2. 単語置換

Prompt:

> I want to read it.

Recognized:

> I want to lead it.

Advice:

> **“read” が “lead” に近く聞こえています。Rの音を少し強めに意識しましょう。**

R/Lのような日本語話者向けカテゴリに接続しやすいです。

### 3. 語尾の欠落

Prompt:

> I liked it.

Recognized:

> I like it.

Advice:

> **語尾の “-ed” が弱くなっています。最後の音を短く添えるように言ってみましょう。**

Clarity系のRuleとして使いやすいです。

### 4. 発話が短すぎる・速すぎる

Expected durationより明らかに短い場合。

Advice:

> **少し速くまとまっています。単語の間を少しだけ空けて話してみましょう。**

Rhythm系に接続できます。

### 5. 特定単語のconfidenceが低い

Prompt内の重要単語だけ低confidence。

Advice:

> **“three” が少し不安定です。舌先を軽く出してTHを作ってみましょう。**

THカテゴリに接続できます。

### 6. 長い無音・詰まり

単語間timestampに大きなgapがある場合。

Advice:

> **途中で少し止まっています。短いフレーズごとに一息で言ってみましょう。**

Fluency / Rhythm向きです。

## Rule adviceの設計イメージ

Ruleはこういうカテゴリにすると良いです。

| Rule Category       | 見るもの            | Advice対象       |
| ------------------- | --------------- | -------------- |
| Missing word        | Promptと認識結果の差分  | Clarity        |
| Substitution        | read→lead など    | R/L, TH, vowel |
| Low confidence word | word confidence | Clarity        |
| Too fast            | duration        | Rhythm         |
| Too slow / pause    | timestamp gap   | Rhythm         |
| Ending dropped      | 語尾差分            | Clarity        |
| Target phoneme word | Prompt辞書        | R/L, TH        |

MVPでは、**Missing / Substitution / Low confidence / Duration** だけで十分です。

# 2. LLMによるアドバイス

LLM adviceは、Apple Speechの結果をそのまま渡して、**自然な一言アドバイスに変換する役割**にすると良いです。

重要なのは、LLMに「音声を評価させる」のではなく、  
**ASR結果・timestamp・confidence・rule候補をもとに、ユーザー向け文言を生成させる**ことです。

## LLMに渡す入力

```
{
  "prompt": "I really like this one.",
  "recognizedText": "I like this one.",
  "durationSec": 2.1,
  "words": [
    { "text": "I", "confidence": 0.95, "start": 0.1, "end": 0.2 },
    { "text": "like", "confidence": 0.82, "start": 0.4, "end": 0.7 },
    { "text": "this", "confidence": 0.78, "start": 0.8, "end": 1.0 },
    { "text": "one", "confidence": 0.91, "start": 1.2, "end": 1.5 }
  ],
  "ruleFindings": [
    {
      "type": "missing_word",
      "target": "really",
      "category": "clarity"
    }
  ],
  "userLevel": "default",
  "tone": "calm, encouraging, concise pronunciation coach"
}
```

## LLMの出力

```
{
  "category": "clarity",
  "target": "really",
  "advice": "“really” が弱くなっています。少し長めに、はっきり出してみましょう。"
}
```

## LLMを使うメリット

Ruleだけだと文言が単調になります。

例えばRuleでは：

> missing word: really

までしか分かりません。

LLMを使うと：

> “really” が弱くなっています。少し長めに、はっきり出してみましょう。

のように、Accent Fitらしいコーチ文言にできます。

## LLMにやらせるべきこと

- Rule findingを自然な一言に変換
- 複数の問題から1つだけ選ぶ
- ユーザーに見せるカテゴリを決める
- トーンを統一する
- 具体的な口・舌・リズムのアクションに変換する

## LLMにやらせない方がよいこと

- 音声そのものの厳密評価
- 音素レベルの断定
- Azureより正確なpronunciation score推定
- 何も根拠がない状態でのR/LやTH判定

つまり、LLMは **Evaluatorではなく Advice Writer / Coach** です。

# 3. おすすめ構成

## ラフ評価

```
Apple Speech
→ transcript / timestamp / confidence
→ Rule Engine
→ LLM Advice Formatter
→ Score + Advice
```

ただし、MVPではさらに分けられます。

### 最小MVP

```
Apple Speech
→ Rule Engine
→ 固定文言Advice
```

### 少し良いMVP

```
Apple Speech
→ Rule Engine
→ LLMで文言生成
```

### 本命

```
Apple Speech
→ Rule Engine
→ LLMが「最も有用な1つのAdvice」を選んで生成
→ 後でAzure詳細評価でPronunciation Map更新
```

***

# 結論

方針としてはこれが良いです。

```
ラフ評価:
Apple Speech + Rule Engine + LLM Advice Writer

詳細評価:
Azure Pronunciation API
```

Ruleは **検出器**、LLMは **コーチ文言生成器**、Azureは **詳細評価器** と分けるのがきれいです。
