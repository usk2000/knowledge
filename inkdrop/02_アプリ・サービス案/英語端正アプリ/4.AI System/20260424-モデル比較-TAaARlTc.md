# モデル比較


# まず整理：今回の評価軸

ユーザー意図を整理すると、モデルに求めたいのはこの2段階です。

## A. ラフ評価向け（MVP / 軽量）

- 発話した内容が合っているか
- だいたい自然なリズムか
- 詰まり / 無音 / 言い直し
- ある程度の confidence

## B. 詳細評価向け（将来）

- 音素単位のズレ
- 母音子音の置換
- stress / rhythm / linking
- nativeとの差分マップ
- どこを直せばよいか

# Pronunciation Mapとは何か

あなたの言う Pronunciation Map はかなり重要です。

例えば：

```
water
W AA T ER

user speech:
W AH D ER
```

差分：

- AA → AH
- T flap化
- stress shift

こういう地図です。

これを出しやすいのは：

1. Azure Pronunciation API
2. wav2vec2 + forced alignment
3. 独自 phoneme model
