# PoC


- **STTと発音評価は分けて考えるべき。**  
  WhisperKit と Azure OpenAI Whisper は強いSTT候補だが、Accent Fitのコア価値である pronunciation scoring / map には Wav2Vec2 系の方が近いです。

- **MVP最短とCore最適は一致しない。**  
  MVPは Whisper 系が速いが、Core差別化は音素系モデルで作る方が自然です。

- **ローカル対応は“iOS先行”が妥当。**  
  WhisperKit は Apple向けで立ち上がりが速い一方、Androidまで同じ実装で揃える設計ではありません。クロスプラットフォーム化は第2段階で考える方が現実的です。

- **PoCではモデル性能だけでなく、UX遅延を測るべき。**  
  Accent Fit は短時間練習アプリなので、精度と同じくらい「録音終了から結果表示までの時間」が重要です。これはプロダクト要件とも整合します


- Whisperは「理解」、Wav2Vec2は「発音」

- MVP最適 ≠ Core最適

- 音素ベース設計は説明可能性が高い

- Accent Fitの価値は「音素差分」にある
