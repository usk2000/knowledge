# 情報設計

情報設計の目的は以下

- 各画面で **何の情報を・どの粒度で・何のために出すか**を明確化
- 「出しすぎ」「曖昧」「将来拡張で破綻」を防ぐ
- 次工程（State設計／API設計／DB設計）にそのまま使える

UIレイアウトではなく、**意味・責務・データ構造寄り**で整理します。

***

# 0. Accent Fit の情報設計 原則（最重要）

### 原則1：評価ではなく「距離」を示す

- ❌ 正誤判定
- ⭕ Fit / Closer / Improvement

### 原則2：1画面＝1意思決定

- Score画面：理解する
- Advice画面：次に何を直すか決める
- Result画面：続けるか／終わるか

### 原則3：数値は「比較用」、言葉は「行動用」

- 数値 → Score / Log
- 言葉 → Advice / Practice

***

# 1. コアドメインモデル（まず概念を固定）

### 発話セッション（Speech Session）

```
SpeechSession
- id
- dateTime
- mode (sentence | free | drill | talk)
- promptId?
- promptSnapshot? (あとでお題が編集された場合を想定)
- audioRef
- fitScore
- breakdown
- keyAdvice
```

### スコア構造

```
FitScore
- overall (0-100)
- rhythm
- stress
- clarity
```

### アドバイス構造

```
FitAdvice
- target (e.g. /r/, endings, stress)
- message (1-2 lines)
- why (optional)
- priority (primary | secondary)
```

# 2. 画面別 情報設計（確定版）

以下、**MVPで必須／後で拡張**を明示します。

***

## 2-1. HOME（Recording）

### 表示する情報

| 情報                             | 必須           | 目的            |
| ------------------------------ | ------------ | ------------- |
| Today’s Fit Score（最新 or 平均）    | ⭕            | 軽い動機づけ        |
| Practice mode（Sentence / Free） | ⭕            | 行動選択          |
| Prompt Text                    | ⭕（Sentence時） | 発話内容の明確化      |
| Sample Audio                   | △            | 正解を「見る」より「聞く」 |
| Tip（短文）                        | △            | 不安軽減          |

### 表示しない情報（重要）

- 過去比較
- 詳細スコア
- 長文説明

→ **即行動に集中させる画面**

***

## 2-2. RESULT（Result Summary）

### 役割

> 「今回の発話をどう捉えるか」を**1秒で理解**

### 表示する情報

| 情報                 | 必須 | 理由       |
| ------------------ | -- | -------- |
| Fit Score（overall） | ⭕  | 定量的な指標   |
| 状態ラベル（Good Fit 等）  | ⭕  | 感情的理解    |
| Quick Insight（1文）  | ⭕  | 次アクション誘導 |
| 保存状態               | ⭕  | 安心感      |

### 重要な制限

- Insightは **1つだけ**
- 技術用語禁止

***

## 2-3. SCORE（Fit Score Detail）

### 役割

> 「どこがズレているか」を**構造的に理解**

### 表示する情報

| 情報                        | 必須 | 備考          |
| ------------------------- | -- | ----------- |
| Overall Score             | ⭕  | Resultとの連続性 |
| Rhythm / Stress / Clarity | ⭕  | 常に3軸で固定     |
| 各軸の短い説明                   | ⭕  | 学習価値        |
| 比較（過去）                    | ❌  | MVPでは出さない   |

### 表示しない情報

- 音素レベル詳細
- 専門用語（formant 等）

→ **分析しすぎないのが成功条件**

***

## 2-4. ADVICE（Fit Advice Detail）

### 役割

> 「次に何をどう直すか」を**即実行可能に**

### 表示する情報

| 情報                   | 必須 | 理由   |
| -------------------- | -- | ---- |
| Primary Target       | ⭕  | 注意集中 |
| Advice Message（1-2行） | ⭕  | 行動指示 |
| Why（任意）              | △  | 納得感  |
| Sample Audio         | △  | 再現性  |
| Retry CTA            | ⭕  | 即改善  |

### ルール

- アドバイスは **最大1つ（MVP）**
- 「直せ」ではなく「試してみよう」

***

## 2-5. PRACTICE（Drills / Fit Talk）

### Drills

| 情報         | 必須 |
| ---------- | -- |
| 対象音 / カテゴリ | ⭕  |
| レベル        | ⭕  |
| 所要時間       | △  |

### Fit Talk

| 情報                   | 必須 |
| -------------------- | -- |
| Topic                | ⭕  |
| AI Prompt            | ⭕  |
| 各ターンの Mini Fit Score | △  |
| セッション全体の Summary     | ⭕  |

※ Fit Talk は **Result / Score / Advice を再利用**する設計

***

## 2-6. LOG（Fit Log）

### 役割

> 「自分は確実に進歩している」と思わせる

### LOG\_LIST

| 情報        | 必須 |
| --------- | -- |
| 日時        | ⭕  |
| Fit Score | ⭕  |
| Mode      | ⭕  |
| ラベル       | △  |

### LOG\_DETAIL

| 情報                | 必須 |
| ----------------- | -- |
| Score + Breakdown | ⭕  |
| Key Advice（当時）    | ⭕  |
| Audio Replay      | ⭕  |
| Retry             | ⭕  |

***

## 2-7. SETTINGS / PROFILE / PRO

### PROFILE

| 情報                   | 必須 |
| -------------------- | -- |
| Goal（Business/Daily） | ⭕  |
| Level                | ⭕  |

### PRO

| 情報               | 必須 |
| ---------------- | -- |
| 制限内容             | ⭕  |
| 価値（Before/After） | ⭕  |

***

# 3. 情報の流れ（最重要）

```
Speech
  ↓
Result（感情）
  ↓
Score（理解）
  ↓
Advice（行動）
  ↓
Retry or Log（継続）
```

この流れを **一度も崩さない**のが設計上の最重要ポイントです。
