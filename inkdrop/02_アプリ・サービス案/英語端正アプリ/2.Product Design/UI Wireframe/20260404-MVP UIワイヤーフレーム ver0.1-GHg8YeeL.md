# MVP UIワイヤーフレーム ver0.1


## 1. 画面方針

MVPは引き続き **1画面中心** です。  
画面遷移ではなく、**同一画面内の状態切替** で学習ループを回します。

## 画面一覧

必要最小限はこの3つです。

1. **Practice Screen**  
   録音、評価、Retry / Skip / Next を行うメイン画面
2. **Session Complete Sheet / Dialog**  
   区切りをつける軽い終了UI
3. **History Screen（任意）**  
   MVPでは省略可能

プロダクトの核は、今回も **Practice Screen** です。  
短く、すぐ始められ、すぐ改善できるという基本方針は変えません。

# 3. Practice Screen 詳細

## 3-1. Header

### 表示要素

- タイトル：`Accent Fit`
- 右上：閉じる
- 補助情報：`Today 3`

### ワイヤー

```
Accent Fit                         Today 3   ✕
```

### UX意図

- 習慣化の気配は出す
- ただし重くしない

***

## 3-2. Prompt Area

### 表示要素

- 1つの英文
- 必要なら補助ラベル `Say this`
- 任意で speaker アイコン

### ワイヤー

```
Say this

“Maybe red leaves never stay.”
```

### 設計ルール

- 2行以内が理想
- 最大でも約10秒で読める長さ
- Promptは結果表示後も残す

# 4. 状態別ワイヤーフレーム

## State A: Idle

### 目的

- 開いた瞬間にすぐ始められる

### 表示

```
Say this

“Maybe red leaves never stay.”

Tap to start recording
```

### ボタン

```
[ Record ]
```

***

## State B: Recording

### 目的

- 録音中であることを明確にする
- 基本は自動停止、必要なら手動停止

### 表示

```
Say this

“Maybe red leaves never stay.”

● Recording...
Speak naturally
```

### ボタン

```
[ Stop ]
```

***

## State C: Processing

### 目的

- 評価中であることだけを短く伝える

### 表示

```
Analyzing your pronunciation...
```

### ボタン

- なし

***

## State D1: Result（threshold未満）

### 条件

- Score < threshold

### 目的

- まず再挑戦を促す

### 表示

```
Fit Score

82

Advice
Pull your tongue back for /r/.
```

### ボタン

```
[ Retry ]

Skip →
```

### 意図

- Primary は Retry
- Skip は離脱・前進のための副導線
- Next は出さない

***

## State D2: Result（threshold以上）

### 条件

- Score >= threshold

### 目的

- 十分できた実感を与え、次へ進ませる

### 表示

```
Fit Score

96

Advice
Nice fit on /r/.
```

### ボタン

```
[ Retry ]   [ Next ]
```

または縦並びなら

```
[ Next ]
[ Retry ]
```

### 意図

- この状態では **Next が正当な主要行動**
- Retry も残すことで、さらに磨きたい人を止めない
- Skip は不要。Next が役割を吸収する

***

# 5. Result状態の分岐ルール

ここが今回の変更の中心です。

## 低スコア時

```
[Retry]
Skip →
```

- 改善ループ優先
- 次へ進む導線は弱くする

## 高スコア時

```
[Retry]   [Next]
```

- 「できたから進む」が自然
- 「もっと詰める」も残す

# 7. ユーザーフロー

## 基本フロー

```
App Open
  ↓
Practice Screen (Idle)
  ↓
Record
  ↓
Recording
  ↓
Processing
  ↓
Result
  ├─ Retry → Recording
  ├─ Skip  → Next Prompt (Idle)        ※ threshold未満時のみ
  └─ Next  → Next Prompt (Idle)        ※ threshold以上時
```

# 8. Button hierarchy の整理

ここは実装前に明確化しておくべき点です。

## threshold未満

- **Primary**: Retry
- **Secondary**: Skip

## threshold以上

候補は2つあります。

### 案A

- **Primary**: Next
- **Secondary**: Retry

これは「合格したら進む」を最も強く表現します。

### 案B

- **Equal weight**: Retry / Next

これは「進んでもいいし、もう一度でもいい」を表現します。

## 推奨

MVPでは **案A** が自然です。

高スコア時は、

```
[ Next ]
[ Retry ]
```

または横並びで Next を視覚的に強くするほうが、UXの意図が明確です。

# 10. MVPで入れないUI

今回も以下は入れません。

- Score内訳
- Pronunciation Map本体
- 難易度設定
- Promptカテゴリ選択
- 詳細履歴
- 過剰な演出

理由は変わらず、MVPの価値が **短い反復ループ** にあるためです。Pronunciation Map は将来の強い価値ですが、初期ワイヤーには載せません。

# 12. このUIで守るべきこと

## 守ること

- Prompt が常に見える
- その時点で何をすべきかが一目でわかる
- threshold によって行動の強さが変わる

## 避けること

- Retry / Skip / Next を常に同時表示する
- 高スコア時にも Skip を残す
- 分析情報を増やしすぎる

***

# Decision

- MVPは **1画面中心** のまま
- Result状態を **threshold未満 / 以上** に分ける
- threshold未満では **Retry 主導 + Skip 副導線**
- threshold以上では **Next を明示**
- 高スコア時の Skip は **Next に吸収**
- Prompt は常時表示
- Advice は音素レベルの短文 + 任意音声補助
