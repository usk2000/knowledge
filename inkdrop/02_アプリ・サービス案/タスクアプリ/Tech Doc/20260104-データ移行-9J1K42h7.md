# データ移行



### アカウント

- アプリでは匿名ログインを必須にしている
- Email等で登録するとき、ユーザが存在するかを確認
  - ユーザが新規の時→リンクしてアプリデータをFirestoreに移動
  - ユーザがすでに存在する時→アプリデータを捨ててFirestoreの情報を使うかどうかをユーザに聞く
    - Yes: ログイン後、ローカル・クラウドのデータを削除、Anonymousアカウントも削除し、Firebaseから情報の取得

### データ移行（ログイン時）

- Precondition: アカウントが存在し、JSONファイルが存在する、かつFirestoreデータが存在しない。
- UserDefaultsでデータ移行が完了したかどうかのフラグを見る
  - UserDefaultsのキーは`is_migrated`, `migration_date`
- ファイルが存在する時、Firestoreを見にいく、
  - settingsを見に行く
- データ移行
  - JSONファイルからFirestoreにアップロード
  - UserDefaultsにデータ移行のフラグを立てる

### 移行後の取り扱い

- FirestoreをSoT(Source of Truth)とする
- ローカルファイルはキャッシュ扱いにするが、Firestoreのローカルキャッシュ機能を利用することを考える


### 備考
