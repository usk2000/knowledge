# C2 前提の実装ロードマップ

## フェーズ0：基盤づくり（環境構築）

**目的:** 最低限の環境と開発フローを固める。

- Firebase プロジェクト作成
- 有効化：
  - Firebase Auth
  - Cloud Firestore
  - Cloud Functions (TypeScript)
  - Hosting（Web を置くなら）
- ローカル開発環境準備
  - `firebase-tools` 導入
  - `firebase emulators:start` で Firestore / Functions のローカル実行
- Client（iOS / Android / Web）
  - Firebase SDK（Auth / Firestore）を導入
  - 共通の環境設定（API エンドポイント / プロジェクトID）

## フェーズ1：素のタスク管理（AIなし）

**目的:** 通常のタスクアプリとして動く状態を作る。

### Backend（Cloud Functions / BFF）

- Firestore スキーマ  
  - `users/{userId}/tasks/{taskId}`  
    - `title: string`
    - `note: string`
    - `status: "todo" | "doing" | "done"`
    - `priority: "low" | "medium" | "high"`
    - `estimateMin: number`
    - `dueAt: Timestamp | null`
    - `createdAt: Timestamp`
    - `updatedAt: Timestamp`

- Task API
  - `GET /tasks`（一覧）
  - `POST /tasks`（作成）
  - `PATCH /tasks/{id}`（編集）
  - `DELETE /tasks/{id}`（削除）
  - `PATCH /tasks/{id}/status`（完了ステータス変更）

- Firestore Security Rules  
  - `request.auth.uid == userId` のチェックを設定


### Client（共通）

- タスク一覧画面
- タスク追加/編集画面
- 完了/未完了トグル
- Firestore 経由で CRUD

## フェーズ2：自然言語 → 構造化タスク（/nl/parse-task）

**目的:** 「自然文 → AIで構造化」機能を作る。

### Backend

- `/nl/parse-task` API
  - Request: `{ text, tz, now, context }`
  - LLM プロンプト設計
  - 出力：  
    ```json
    {
      "task": { ...構造化タスク... },
      "rationale": "...理由...",
      "confidence": 0.92
    }
    ```
- 日付・時刻を tz に基づき正規化

### Client

- 「自然文入力欄」を作成
- `/nl/parse-task` 呼び出し
- 返ってきた草案をフォームに反映し、ユーザーが編集して保存

## フェーズ3：ベクトルDB連携（Qdrant）＋セマンティック検索

**目的:** 意味検索をできるようにする。

### Backend – ベクトル化

- Firestore Trigger
  - `onCreate`: タスク作成 → Embedding → Qdrant `upsert`
  - `onUpdate`: title/note 変更時 → 再埋め込み → Qdrant `upsert`
  - `onDelete`: Qdrant `delete`
- Embeddings API（OpenAIなど）を実装
- Qdrant クライアント実装
  - `upsert(id, vector, payload)`
  - `delete(id)`


### Backend – Search API

- `POST /semanticSearch`
  - `query` を Embedding へ
  - Qdrant `search`（filter:userId）
  - Firestore から taskId リストを詳細取得
  - 類似度付きで返却

### Client

- セマンティック検索バーを追加
- 結果一覧画面

## フェーズ4：今日のプラン（推薦）＋カレンダー連携

**目的:** 「今日やるタスク」を AI が提案する。

### Backend – Plan API

- `GET /plan/today`
  - Firestore から未完了タスク取得
  - カレンダー API から Free/Busy 取得
  - スコアリング（締切・優先度・見積×空き枠・延期回数）
  - 任意で Qdrant を RAG として参照
  - LLM に「並べ替え＋理由説明」を依頼
  - ランク済みリストを返却

### Client

- 「今日のプラン」画面
- 承認すると予定ブロック作成（カレンダー
