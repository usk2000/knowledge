## それぞれの画面ごとのNavHostControllerから画面を作成する

```kotlin
@Composable
fun ContentNavHost(navController: NavHostController) {
    NavHost(
        navController = navController,
        startDestination = "contents"
    ) {
        composable("contents") {
            ContentListScreen { id ->
                println("ID: $id")
                navController.navigate("contents/${id}")
            }
        }
        composable(
            route = "contents/{id}",
            arguments = listOf(navArgument("id") { type = NavType.StringType })
        ) { backStackEntry ->
            val id = backStackEntry.arguments?.getString("id")!!
            ContentDetailScreen(
                id = id,
                onBack = { navController.popBackStack() }
            )
        }
    }
}
```

# 方安全な方法
## 1️⃣ まとめ方の例（object で名前空間化）

```
import kotlinx.serialization.Serializable

object ContentsRoutes {

    @Serializable
    object List

    @Serializable
    data class Detail(val id: String)
}
```

- 継承なし

- ただの「グルーピング用の入れ物」として `object ContentsRoutes` を使うだけ

***

## 2️⃣ NavHost / NavGraph の書き方

```
import androidx.navigation.NavHostController
import androidx.navigation.NavGraphBuilder
import androidx.navigation.compose.composable
import androidx.navigation.toRoute

fun NavGraphBuilder.contentsNavGraph(
    navController: NavHostController,
) {
    // 一覧
    composable<ContentsRoutes.List> {
        TopListScreen(
            onItemClick = { id ->
                navController.navigate(ContentsRoutes.Detail(id))
            }
        )
    }

    // 詳細（id 付き）
    composable<ContentsRoutes.Detail> { backStackEntry ->
        val args = backStackEntry.toRoute<ContentsRoutes.Detail>()

        TopDetailScreen(
            id = args.id,
            onBack = { navController.popBackStack() }
        )
    }
}
```

***

## 3️⃣ NavHost の startDestination

```
@Composable
fun ContentsRoot() {
    val navController = rememberNavController()

    NavHost(
        navController = navController,
        startDestination = ContentsRoutes.List
    ) {
        contentsNavGraph(navController)
    }
}
```

# Serializationを使うために
# ✅ 1. `libs.versions.toml` に追加する

```
[versions]
kotlin = "2.0.0"
serialization = "2.0.0"  # Kotlin と同一にするのが安全

[plugins]
kotlin-serialization = { id = "org.jetbrains.kotlin.plugin.serialization", version.ref = "serialization" }
```

> ※ Compose Navigation 2.8+ と組み合わせるなら Kotlin 2.0 系が推奨です。

***

# ✅ 2. モジュール（app / shared など）の `build.gradle.kts`にプラグインを適用

```
plugins {
    alias(libs.plugins.kotlin.serialization)
}
```

Android モジュールなら：

```
plugins {
    alias(libs.plugins.android.application)
    alias(libs.plugins.kotlin.android)
    alias(libs.plugins.kotlin.serialization)
}
```

KMP モジュールなら：

```
plugins {
    alias(libs.plugins.kotlin.multiplatform)
    alias(libs.plugins.kotlin.serialization)
}
```

Note: モジュールのpluginsに入れる

***

# ❗ 3. 依存関係（kotlinx-serialization-json）も必要

`toRoute()` の内部で serialization を使うため、JSON の依存も入れます。

`libs.versions.toml`：

```
[libraries]
kotlinx-serialization-json = { group = "org.jetbrains.kotlinx", name = "kotlinx-serialization-json", version = "1.6.3" }
```

`build.gradle.kts`：

```
dependencies {
    implementation(libs.kotlinx.serialization.json)
}
```
